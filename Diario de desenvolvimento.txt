Sessao 4 - 21 de marco de 2026
Objetivo: Melhorar o rastreamento do corpo (skeleton tracking), especialmente dos bra\u00e7os.

Problemas identificados:
- O esqueleto nao conseguia ler corretamente os movimentos dos bra\u00e7os (cotovelos e pulsos).
- Thresholds de confianca muito altos (0.10-0.12) para o PoseNet, que naturalmente retorna
  valores baixos para os bra\u00e7os, especialmente durante movimentos rapidos.
- Sistema de tracking historico fraco (TRACKED_POINT_HOLD = 35 frames) insuficiente para
  rastrear bra\u00e7os que se movem rapidamente.
- Falta de suavizacao (smoothing) - os pontos dos bra\u00e7os tremulavam muito entre frames.
- Detecao de bloqueio com bra\u00e7o (doesForearmBlock) muito severa - bracos raramente
  conseguiam bloquear obstaculos.

Solucoes implementadas:
1. Aumentado TRACKED_POINT_HOLD de 35 para 60 frames (~1 segundo de retenção).
2. Adicionado sistema de suavizacao EMA (Exponential Moving Average) com factor 0.65
   especificamente para os bra\u00e7os (indices 7-10).
3. Reduzidos thresholds de confianca:
   - Bra\u00e7os: de 0.10-0.12 para 0.05-0.08 (muito mais lenientes)
   - Corpo: mantidos em 0.10-0.12 para estabilidade
4. Melhorada deteccao de colisao de bra\u00e7os (doesForearmBlock):
   - Reduzido threshold de 0.10 para 0.05
   - Aumentada tolerancia de deteccao (hitPad de 10 para 14)
5. Melhorada deteccao de T-pose (isArmInTPose):
   - Thresholds de ombro: 0.10 (mantido)
   - Thresholds de pulso: 0.08 → 0.06
   - Thresholds de cotovelo: 0.10 → 0.06
   - Aumentada tolerancia para movimento vertical dos bra\u00e7os (maxYDelta 60→80)

Tecnologia aplicada:
- EMA (Exponential Moving Average) para reducao de jitter
- Thresholds adaptativos (diferentes consoante o tipo de ponto corporal)
- Historico de pontos para colmatar gaps temporarios na deteção

Resultado esperado:
- Braços devem ser rastreados mais consistentemente
- Bloqueio de obstaculos com os bra\u00e7os deve funcionar melhor
- Deteção de T-pose debe ser mais robusta mesmo com tremulação
