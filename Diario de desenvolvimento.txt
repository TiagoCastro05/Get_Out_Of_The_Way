Sessao 5 - 21 de marco de 2026
Objetivo: Melhorar o rastreamento do corpo (skeleton tracking), especialmente dos bra\u00e7os.

Problemas identificados:
- O esqueleto nao conseguia ler corretamente os movimentos dos bra\u00e7os (cotovelos e pulsos).
- Thresholds de confianca muito altos (0.10-0.12) para o PoseNet, que naturalmente retorna
  valores baixos para os bra\u00e7os, especialmente durante movimentos rapidos.
- Sistema de tracking historico fraco (TRACKED_POINT_HOLD = 35 frames) insuficiente para
  rastrear bra\u00e7os que se movem rapidamente.
- Falta de suavizacao (smoothing) - os pontos dos bra\u00e7os tremulavam muito entre frames.
- Detecao de bloqueio com bra\u00e7o (doesForearmBlock) muito severa - bracos raramente
  conseguiam bloquear obstaculos.

Solucoes implementadas:
1. Aumentado TRACKED_POINT_HOLD de 35 para 60 frames (~1 segundo de retenção).
2. Adicionado sistema de suavizacao EMA (Exponential Moving Average) com factor 0.65
   especificamente para os bra\u00e7os (indices 7-10).
3. Reduzidos thresholds de confianca:
   - Bra\u00e7os: de 0.10-0.12 para 0.05-0.08 (muito mais lenientes)
   - Corpo: mantidos em 0.10-0.12 para estabilidade
4. Melhorada deteccao de colisao de bra\u00e7os (doesForearmBlock):
   - Reduzido threshold de 0.10 para 0.05
   - Aumentada tolerancia de deteccao (hitPad de 10 para 14)
5. Melhorada deteccao de T-pose (isArmInTPose):
   - Thresholds de ombro: 0.10 (mantido)
   - Thresholds de pulso: 0.08 → 0.06
   - Thresholds de cotovelo: 0.10 → 0.06
   - Aumentada tolerancia para movimento vertical dos bra\u00e7os (maxYDelta 60→80)

Tecnologia aplicada:
- EMA (Exponential Moving Average) para reducao de jitter
- Thresholds adaptativos (diferentes consoante o tipo de ponto corporal)
- Historico de pontos para colmatar gaps temporarios na deteção

Resultado esperado:
- Bra\u00e7os devem ser rastreados mais consistentemente
- Bloqueio de obstaculos com os bra\u00e7os deve funcionar melhor
- Deteção de T-pose debe ser mais robusta mesmo com tremulação

3. Thresholds minimos reduzidos DRASTICAMENTE:
   - Braços: 0.05 → 0.02 (praticamente aceita qualquer confiança)
   - Colisao de bra\u00e7os: 0.05 → 0.02

4. Tolerancia de T-pose muito maior:
   - maxYDelta: 80 → 120 pixels (permite braços bem acima da cabeça)
   - Zona de cotovelo: 88 → 135 pixels
   - Espaço entre punho e cotovelo: 40 → 50 pixels
5. Melhorado drawLimbWithFallback:
   - Simplificada logica de verificacao
   - Usa threshold 0.02 direto para bra\u00e7os
6. Aumentado hitPad em doesForearmBlock: 14 → 18 pixels

Resultado esperado:
- Braços levantados devem ser detectados e renderizados

Sessao 8 - 21 de marco de 2026 (CORRECOES CRITICAS)
Objetivo: Resolver 2 problemas críticos:
1. Esqueleto não reconhecia bem posições quando corpo estava longe (inteiro visível)
2. Obstáculos bloqueados não eram removidos, causando dano em outras partes do corpo

PROBLEMA 1 - ESQUELETO/T-POSE CUANDO CORPO LONGE:

Causa raiz: Thresholds de confiança muito altos!
- Quando o utilizador está longe (corpo todo visível até joelho), a confiança do PoseNet é muito baixa (~0.01-0.05)
- Anterior: Thresholds 0.15-0.20 (rejeitava a maioria dos keypoints)
- Resultado: Braços "desapareciam" ou mostravam em posição errada

Soluções implementadas:

A) isTPose - Reduzir thresholds DRASTICAMENTE:
  - Ombros: 0.2 → 0.05 (4x mais leniente!)
  - Pulsos: 0.1 → 0.03 (3x mais leniente!)
  - Ancas: 0.2 → 0.05 (4x mais leniente!)
  
B) isTPose - Critérios mais flexíveis:
  - bodyUpright: 40px → 30px (mais tolerante com corpo ligeiramente dobrado)
  - MIN_ARM_REACH: 60px → 40px (aceita braços dobrados/relaxados)
  - Tolerância vertical: 150px → 200px (aceita pulsos mais variados)
  
C) drawSkeleton - Thresholds de renderização muito mais baixos:
  - Torso: 0.15 → 0.05 (3x mais sensível)
  - Braços: 0.1 → 0.02 (5x mais sensível!)
  - Pernas: 0.15 → 0.05 (3x mais sensível)
  - Pontos: 0.15 → 0.05 (3x mais sensível)
  
D) Funções de colisão com thresholds drasticamente reduzidos:
  - canArmBlock: 0.08 → 0.03
  - canLegBlock: 0.15 → 0.05
  - hasBodyCollision: 0.15 → 0.05

Resultado: Esqueleto agora renderiza corretamente mesmo quando corpo está longe!
Bra\u00e7os em T-pose aparecem na posição correta, mesmo que relaxados ou dobrados.

PROBLEMA 2 - OBSTÁCULOS BLOQUEADOS NÃO DESAPARECEM:

Causa raiz: checkCollision retornava false para bloqueio (continua = sem remover)
- canArmBlock() bloqueava com sucesso → checkCollision retorna false
- updateObstacles() vê false → não remove obstáculo
- Obstáculo continua a passar pelo corpo → dano desnecessário

Solução implementada:

A) Modificar checkCollision para retornar 3 estados:
  - "blocked" = Bloqueado com braço/perna (remove sem dano)
  - true = Colisão com corpo (remove com dano)
  - false = Sem colisão (deixa passar)
  
B) updateObstacles com lógica corrigida:
  ```javascript
  let collisionResult = checkCollision(ob);
  if (collisionResult === "blocked") {
    // Remove o obstáculo
    obstacles.splice(i, 1);
    continue; // Sem dano!
  } else if (collisionResult === true) {
    // Colisão com corpo
    loseLife();
    obstacles.splice(i, 1);
    continue;
  }
  ```

Resultado: Obstáculos bloqueados desaparecem imediatamente sem causar dano!

RESUMO DAS MUDANÇAS:

| Componente | Mudança | Impacto |
|-----------|---------|--------|
| T-pose thresholds | 0.2 → 0.05 avg | 4x mais leniente |

| T-pose critérios | Mais flexíveis | Aceita braços relaxados |
| Colisão thresholds | ~0.08-0.15 → 0.03-0.05 | 2-3x mais sensível |
| Remoção obstáculos | Não removia bloqueados | Agora remove corretamente |
